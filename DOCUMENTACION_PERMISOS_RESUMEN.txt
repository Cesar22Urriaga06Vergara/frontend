📋 RESUMEN RÁPIDO DEL SISTEMA:

  ESTRUCTURA:
    • 8 roles (superadmin, admin, recepcionista, cliente, cafeteria, lavanderia, spa, room_service)
    • Matriz de permisos en composables/usePermissions.ts
    • Cada rol tiene array de permisos strings

  API usePermissions():
    const { can, canAny, canAll, hasRole, hasAnyRole } = usePermissions()
    
    can('permiso')          ← ¿Tiene THIS permiso?
    canAny('a', 'b')        ← ¿Tiene AL MENOS UNO?
    canAll('a', 'b')        ← ¿Tiene TODOS?
    hasRole('admin')        ← ¿Es este rol?
    userPermissions.value   ← Array de permisos

  3 NIVELES DE CONTROL:
    Nivel 3: definePageMeta({ roles: [...] })    ← Hard constraint
    Nivel 2: useRoleNavigation() → items           ← Invisible
    Nivel 1: <btn v-if="can(...)">                ← Best UX

  AGREGAR NUEVO PERMISO (3 PASOS):
    1. Agregar string en ROLE_PERMISSIONS[rol]
    2. Usar en template: <btn v-if="can('permiso')">
    3. (Opcional) Proteger ruta: roles: [UserRole.ADMIN]

📁 ARCHIVOS CLAVE DEL CÓDIGO:

  composables/usePermissions.ts
    ├─ ROLE_PERMISSIONS = { admin: [...], recepcionista: [...] }
    ├─ can(permiso) → boolean
    ├─ hasRole(rol) → boolean
    └─ userPermissions → computed

  composables/useRoleNavigation.ts
    ├─ navigationSections (items diferentes por rol)
    ├─ roleLabel, roleColor, roleIcon (presentación)
    └─ defaultRoute (ruta por defecto del rol)

  middleware/auth.ts
    └─ Verifica: ¿token + user existen?

  middleware/role.ts
    ├─ Lee meta.roles de la ruta
    ├─ Compara con user.role
    └─ Redirige si no coincide

  utils/constants.ts
    ├─ ROLE_LABELS (etiquetas legibles)
    ├─ ROLE_COLORS (colores para UI)
    ├─ ROLE_ICONS (iconos)
    └─ ROLE_DEFAULT_ROUTE (ruta por rol)

  stores/auth.ts
    ├─ user.role (tu rol actual)
    └─ userRole getter (normalizado a minúsculas)

  types/auth.ts
    └─ enum UserRole { ADMIN, SUPERADMIN, ... }

⚠️  IMPORTANTE:

  ❌ Frontend permisos ≠ Seguridad real
  ✅ Backend DEBE validar también con @Roles(UserRole.ADMIN)

  Cualquiera puede hacer en DevTools:
    localStorage.setItem('auth', ...)
    authStore.user.role = 'admin'

  Pero no puede falsificar:
    Token JWT (es firmado en backend)
    DB record (es verificado en backend)
