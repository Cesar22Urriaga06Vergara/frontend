# Documentacion Tecnica Del Frontend

Esta carpeta conserva documentacion operativa, auditorias y notas de evolucion
del frontend. No forma parte del runtime de Nuxt, pero sirve como referencia
tecnica del proyecto.

## Estructura

```text
docs/
  audits/      Auditorias, diagnosticos, riesgos y brechas detectadas.
  planning/    Roadmaps, checklist y funcionalidades pendientes.
  operations/  Flujos operativos visibles para usuarios del sistema.
  domain/      Reglas de dominio, facturacion, roles, permisos y KPIs.
  changelog/   Cambios tecnicos, estabilizaciones y mejoras aplicadas.
```

## Criterio De Conservacion

Estos archivos se mantienen porque contienen decisiones tecnicas, pendientes,
riesgos o reglas operativas que todavia pueden ser utiles para soporte,
validacion y desarrollo. Si un documento se vuelve obsoleto, primero debe
consolidarse en el README principal o en una guia vigente antes de eliminarlo.
